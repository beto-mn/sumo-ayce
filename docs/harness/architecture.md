## Why this file exists

The base harness playbook assumes architecture lives here (in `docs/`).
Since this project uses spec-kit, the source of truth moved to the
constitution. This stub exists only so `init.sh` doesn't fail when
verifying base files.

## How to read the architecture

1. Open `.specify/memory/constitution.md`.
2. Read the preamble and all articles (Article I, II, ...).
3. Pay attention to principles marked as `NON-NEGOTIABLE` — those are
   the Phase -1 gates that appear in every `plan.md`.
